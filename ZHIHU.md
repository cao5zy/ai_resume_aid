# 我用 AI 做了个免费简历工具，专门帮那些不太会「包装」自己的人

> 你的简历不需要华丽，但要被看见

---

## 引子

去年秋天，一位朋友发来一份简历让我帮忙看看。他四十出头，在工厂做了十几年设备维护，想换个环境。打开他的简历，密密麻麻好几段，从第一份工到最近一份工，按时间列了一遍，没有重点，没有亮点。

我帮他改完发回去，他回了条消息：

「原来我做过这么多事。」

就这么简单一句话，让我记到现在。

后来我开始注意这件事。在中文互联网上搜"简历优化"、"简历修改"，出来的全是面向白领的服务——什么 STAR 法则、什么高管简历包装、什么 99 元一次的专业修改。这些东西好不好？好。但谁在用？在写字楼里上班的人。

那工厂的工人、送外卖的小哥、视障的朋友、从农村出来打工的年轻人呢？他们不跳槽、不找工作吗？他们当然也找。只是他们的简历可能就几句话——「在电子厂干了三年」「做过保洁」「帮人开过车」——他们不知道怎么写，也不知道从哪里问。

这个想法在心里搁了很久。今年春末，我腾出时间，把它做出来了。

## 为什么是「简历」？为什么不是「又一款简历工具」？

市面上 AI 简历工具很多，但我观察到一个现象：**绝大多数工具在服务同一拨人——已经有不错表达能力、有一定职场经验的白领。** 它们的优化逻辑是「让好变得更好」，而不是「让不知道怎么写的人写出来」。

而我想服务的是另外四类人：

- **残障求职者** — 有实实在在的技能，但简历里往往花大量篇幅解释身体状况，反而把能力挤到边角
- **45 岁以上的大龄求职者** — 十几年实操经验，简历却写得像流水账，还经常被"35 岁门槛"先入为主地筛掉
- **乡村务工青年** — 做过好几份工，但经历零散，不知道哪些值得写、怎么写
- **偏远地区的贫困应届生** — 没有光鲜的实习经历，没有学长内推，求职信息闭塞

这四类人占求职者很大比例，但几乎没有工具专门为他们设计过。

所以「AI微光求职」的核心理念很简单：**不追求帮你把简历变得多华丽，而是帮你把已有的东西说清楚。**

## 它是怎么工作的

用起来很简单——三步走：

**第一步** 选择你属于哪类人群。目前支持障碍者和 45 岁以上两类，后续计划拓展更多。

**第二步** 输入简历内容。两种方式：直接粘贴文本，或者上传已有的 PDF 简历。

![输入界面截图 — 显示粘贴文本和上传PDF两种方式]
*（此处替换为你截取的输入界面截图：textarea 和文件上传两种输入方式）*

**第三步** 点击「开始优化」，等几十秒到几分钟。

后台做的事其实挺多的：AI 会根据你选择的人群，用不同的策略来优化。比如障碍者的简历，会弱化对身体状况的描述，把技能和工作成果往前放；大龄求职者的简历，会注重凸显实操经验，淡化与年龄相关的信息。

优化完成后，你会看到：

- 优化前后的**对比**，方便对照
- 一段**鼓励的话**——这不是心理鸡汤，是希望大家在求职季能多一点点信心
- 匹配你群体的**包容性岗位推荐**（通过知乎搜索 API 获取）
- **导出 PDF** 和**下载 Markdown** 两种保存方式

## 技术上的选择

作为个人开发者，我选技术栈的原则很朴素：够用、免费、长期维护。

- **前端**：Vue 3 + TypeScript + Element Plus
- **后端**：Rust + Axum
- **AI 能力**：知乎直答 Agent 接口
- **岗位信息**：知乎搜索 API
- **PDF 处理**：Rust 原生库

选 Rust 做后端可能有点「重」，但好处是编译完一个二进制文件就能跑，部署成本极低。

全栈部署在云服务上，项目完全免费，不需要登录，不需要注册，没有任何付费功能。

## 为什么个人开发者要做公益工具

一个经常被问的问题：你做这个图什么？

说实话，我做之前也没想清楚。但做的时候有一些瞬间让我觉得这件事值得：

- 给一位视障朋友演示，他第一次用键盘全程操作完成简历优化，没有借助鼠标
- 内测群里有人说「这个页面字真大，不用戴老花镜了」——其实我们只是加了个大字体切换按钮
- 有用户反馈：你们那个鼓励的话，比简历本身还暖

这些反馈让我意识到一件事：**技术的价值不只在「解决大问题」，有时候也在「回应小需求」。** 这些需求太具体了，具体到大型商业公司不太可能专门去满足——因为用户量不够大、商业回报不够高。

那谁来满足呢？个人开发者可以。

不需要融资，不需要对 KPI 负责，不需要日活月活转化率。只需要一台电脑、一些时间、一些诚意。做出来的东西哪怕只帮到几十个人，也是实实在在的价值。

这个项目取名「AI微光」，是因为我希望它能成为求职路上的一点点光源——不够亮，但能照到一些人。

## 写在最后

这个项目是 2026 人文季·公益单元的一份投稿。主题是「用 AI 重新看见人」。

我觉得这个主题说得很对。AI 可以润色文字、匹配岗位、生成话术，但它不会天然地去关心一个工厂工人的简历该怎么写、一个视障者怎么用键盘完成操作。这些东西需要有人去想，然后写到代码里。

好的 AI 应用，不是用 AI 取代人，而是用 AI 去补上那些被忽略的缺口。
