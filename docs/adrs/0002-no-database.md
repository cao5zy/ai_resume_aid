# ADR 0002: 无服务端持久化

## 状态

已采纳（2026-07-01）

## 上下文

项目定位为完全免费的公益工具，目标用户是弱势就业群体。需要决定是否将用户上传的简历数据持久化到数据库。

## 决策

不引入数据库。所有用户数据处理仅在内存中完成，请求完成后不保留任何用户数据。

## 理由

- **隐私安全**：用户简历包含姓名、联系方式、工作经历等敏感个人信息。不持久化意味着无数据泄露风险，也不需考虑数据合规问题
- **部署简化**：无需配置数据库（MongoDB/PostgreSQL/Redis），部署就是一个 Rust 二进制 + 静态文件，降低维护成本和服务器要求
- **公益定位**：项目无商业模式，不收集用户数据，符合公益透明化要求
- **功能需求匹配**：所有功能（文本优化、PDF 导出、岗位搜索）均为一次性请求-响应模式，无需要跨请求关联的状态

## 替代方案

- 考虑过用 SQLite 做使用统计（匿名），但 MVP 阶段价值不匹配成本
- 后续可考虑纯前端统计（如通过 GitHub Pages 的访问计数），无需后端数据库

## 后果

- 无法做用户数据回溯（用户每次都需要重新粘贴/上传简历）
- 无法做使用量统计（可通过 nginx 日志或前端埋点弥补）
- 后端重启不丢失任何数据（因为本来就没有）
